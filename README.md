## **Домашнее задание 20.1 Работа с ORM в Django**

1. Подключено СУБД PostgreSQL для работы в проекте. Для этого:  
-Создана базу данных в ручном режиме.  
-Внесены изменения в настройки подключения.  

2. В приложении каталога созданы модели:  
-Product,  
-Category.  
Описаны для них начальные настройки.  

3. Для каждой модели описаны следующие поля:
Product: наименование, описание, изображение (превью), 
категория, цена за штуку, дата создания,
дата последнего изменения.  
Category: наименование, описание.  
Для поля с изображением добавлены соответствующие настройки в проект, 
а также установлена библиотека для работы с изображениями Pillow.  

4. Перенесено отображение моделей в базу данных с помощью инструмента миграций.  
Для этого:
-Создана миграции для новых моделей.  
-Применены миграции.  
-Внесены изменения в модель категорий, добавлено поле created_at,  
применено обновление структуры с помощью миграций.
-Откатили миграцию до состояния, когда поле created_at
 для модели категории еще не существовало, и удалена лишня миграция.

5. Для моделей категории и продукта настроено отображение в административной панели.  
Для категорий выведены id и наименование в список отображения,  
а для продуктов выведены в список id, название, цена и категория.  
При этом интерфейс вывода продуктов настроен так, чтобы можно было результат отображения фильтровать по категории,  
а также осуществлять поиск по названию и полю описания.  

6. Через инструмент shell заполнен список категорий, а также выберан список категорий, применив произвольные рассмотренные фильтры.  
В качестве решения приложены скриншоты.  
Сформированы фикстуры для заполнения базы данных.  
Написана кастомная команда, которая умеет заполнять данные в базу данных, 
при этом предварительно зачистив ее от старых данных.
